Clip the image to the hurricane#

From the band 13 plot it looks like, in x,y coordinates, the hurricane is between 2.4e6 -> 3.e6 meters in the y dimension and -1.5e6 -> -0.5e6 meters in the x dimension.

Use that to slice the dataset using isel

hit_x = np.logical_and(rioC.x > -1.5e6, rioC.x < -0.5e6)
hit_y = np.logical_and(rioC.y > 2.4e6, rioC.y < 3.0e6)
slice_michael = rioC.isel(indexers={'x':hit_x,'y':hit_y})
slice_michael.dims

Reproject from geostationary to UTM#

The geostationary crs is a little unusual ‚Äì in some cases (to compare with landsat or other satellites, mapping) we might want to reproject from that crs to something standard like universal transfer mercator.

rioxarray.estimate_utm_crs() gives you its best guess, based on the image corners, about which UTM zone we‚Äôre in. It correctly says that Michael is in UTM zone 16N

utm_crs = slice_michael.rio.estimate_utm_crs()
utm_crs

CRS.from_epsg(32616)

Reproject to UTM Zone 16N#

Reprojecting is much simpler that the resampling of ungridded lons/lats we did for the Modis swaths in week 5. rioxarray.rio.reproject does it in a single step

slice_utm16N = slice_michael.rio.reproject(utm_crs)
slice_utm16N.dims

Frozen({'x': 491, 'y': 317})

Note that the reprojection has increased the pixel size to 2.3 km x 2.3 km and changed the array shape to from ‚Äòx‚Äô: 499, ‚Äòy‚Äô: 299 to ‚Äòx‚Äô: 491, ‚Äòy‚Äô: 317
